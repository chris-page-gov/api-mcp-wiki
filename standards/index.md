# Standards & specifications

Protocols, formats and specs cited in the timeline — each verified against its primary source.

- [HTTP/0.9 (1991)](http-0-9.md) — The original, minimal HTTP: a request for a document followed by the document itself.
- [HTTP/1.0 — RFC 1945 (May 1996)](http-1-0.md) — First documented HTTP: MIME-like messages, headers, metadata, content types, richer semantics.
- [XML 1.0 (W3C, Feb 1998)](xml.md) — Structured, extensible markup underpinning SOAP/WSDL-era web services.
- [HTTP/1.1 — RFC 2068 / 2616 / 9112](http-1-1.md) — Made the Web scalable: caching, persistent connections, intermediaries, content negotiation.
- [SOAP 1.1 (W3C Note, May 2000)](soap.md) — Message- and operation-oriented web-service protocol.
- [REST — Fielding dissertation (2000)](rest.md) — Architectural style of constraints, not a synonym for JSON endpoints.
- [WSDL 1.1 (W3C Note, March 2001)](wsdl.md) — Operation-oriented service-definition language for web services.
- [OAuth 2.0 — RFC 6749 (2012)](oauth-2-0.md) — Delegated authority: limited access to an HTTP service without sharing the password.
- [GraphQL (open-sourced 2015)](graphql.md) — Typed query language letting consumers request exactly the data they need.
- [gRPC](grpc.md) — High-performance RPC over Protocol Buffers, suited to internal service-to-service and streaming.
- [HTTP/2 — RFC 7540 (2015)](http-2.md) — Binary framing and multiplexing while retaining HTTP application semantics.
- [JSON — RFC 4627 → RFC 8259 (STD 90)](json.md) — Lightweight data interchange; current standard is RFC 8259.
- [CloudEvents 1.0 (CNCF, Oct 2019)](cloudevents.md) — Common metadata envelope for events across platforms and transports.
- [HTTP Semantics — RFC 9110 (2022)](http-semantics.md) — Separates durable HTTP semantics from the wire formats that carry them.
- [HTTP/3 — RFC 9114 (2022)](http-3.md) — Maps HTTP semantics onto QUIC, with independent streams.
- [Model Context Protocol (MCP)](mcp.md) — Emerging standard connecting LLM applications to tools and data sources.
- [OAuth 2.0 Security BCP — RFC 9700 (Jan 2025)](oauth-2-0-security-bcp.md) — Current best practice; deprecates modes now considered insecure.
- [Agent2Agent (A2A)](a2a.md) — Emerging protocol for communication and collaboration between AI agents.
- [OpenAPI (3.0 2017 · 3.2 Sept 2025)](openapi.md) — Vendor-neutral, machine-readable description format for HTTP APIs.
- [AsyncAPI (3.0 2023 · 3.1 Jan 2026)](asyncapi.md) — Describes message-driven and event-oriented APIs.
- [Arazzo 1.1 (2026)](arazzo.md) — Describes sequences of API calls and their dependencies (workflows).
