# Glossary

Plain-English definitions of the terms that appear in the concept descriptions. Each links to the concepts it explains.

- [Automated decision](automated-decision.md) — A decision made by automated processing, with little or no human involvement.
- [Bearer token](bearer-token.md) — A credential where mere possession grants access; if stolen it can be replayed.
- [Binary framing](binary-framing.md) — Encoding messages as compact binary frames rather than plain text.
- [Channel](channel.md) — A named pathway (like a topic or queue) that messages flow through.
- [Content negotiation](content-negotiation.md) — Client and server agree on the best representation (format/language) of a resource.
- [Delegated authority](delegated-authority.md) — Granting a third party limited permission to act on your behalf without sharing credentials.
- [Event envelope](event-envelope.md) — A common metadata wrapper around an event so different systems handle it uniformly.
- [Event-driven](event-driven.md) — Producers emit events (notifications that something happened) and consumers react.
- [Executable contract](contract.md) — A machine-readable API description that documents it and drives validation, testing and code generation.
- [Human-in-the-loop](human-in-the-loop.md) — A control requiring a person to review or approve before an automated action proceeds.
- [Idempotency](idempotency.md) — An operation that can be repeated safely with the same effect as doing it once.
- [Meaningful human involvement](meaningful-human-involvement.md) — Genuine human review or agency in a decision, not a rubber-stamp.
- [Media type](media-type.md) — A label such as application/json that identifies the format of content.
- [Message-driven](message-driven.md) — Components communicate by sending discrete messages rather than calling each other directly.
- [Multiplexing](multiplexing.md) — Carrying many independent streams over one connection at the same time.
- [Open standard](open-standard.md) — A specification anyone may implement, typically vendor-neutral and royalty-free.
- [Operation](operation.md) — A specific send or receive action on a channel.
- [Provenance](provenance.md) — The recorded origin and history of data or an action.
- [QUIC](quic.md) — A modern UDP-based transport underpinning HTTP/3 with independent streams.
- [RPC](rpc.md) — Calling a function on another machine as if it were local.
- [Redress](redress.md) — The route for a person to obtain explanation, human review and appeal.
- [Request–response](request-response.md) — Call-and-wait: a client sends a request and waits for the server's reply.
- [Schema](schema.md) — A formal description of the structure of data, used to validate and generate code.
- [Semantic interoperability](semantic-interoperability.md) — Agreement not just on data format but on what the data means.
- [Standards body](standards-body.md) — An organisation that develops and maintains technical standards.
- [Statelessness](statelessness.md) — Each request carries everything needed; the server keeps no client session between requests.
- [Workflow](workflow.md) — An ordered sequence of API calls with dependencies that together achieve an outcome.
- [Workload identity](workload-identity.md) — A verifiable identity for software or workloads, not humans.
