# Timeline

The era-by-era narrative (reviewed; corrections folded in).

- [1989–1993 — The Web and the earliest HTTP](1989-1993-web-and-earliest-http.md) — CERN, the first browser/server, and HTTP/0.9; royalty-free release in 1993.
- [1996 — HTTP/1.0 formally documented](1996-http-1-0.md) — RFC 1945 adds headers, metadata, content types and richer semantics.
- [1997–1999 — HTTP/1.1 makes the Web scalable](1997-1999-http-1-1.md) — Caching, persistent connections, intermediaries, content negotiation.
- [1998–2001 — Two traditions: web services and REST](1998-2001-web-services-and-rest.md) — XML/SOAP/WSDL vs REST — overlapping problems at different levels.
- [2000–2006 — APIs become platforms](2000-2006-apis-as-platforms.md) — eBay's APIs and JSON; the resource-oriented HTTP+JSON model emerges.
- [2012 — Delegated authority and API-first government](2012-delegated-authority-api-first.md) — OAuth 2.0 and legislation.gov.uk's API-first architecture.
- [2014–2017 — The modern API ecosystem forms](2014-2017-modern-api-ecosystem.md) — HTTP/2, OpenAPI, and architectural pluralism (GraphQL, gRPC, AsyncAPI).
- [2018–2019 — UK standardisation and event interoperability](2018-2019-uk-standardisation-events.md) — GDS API standards, Open Standards Board OpenAPI 3, CloudEvents 1.0.
- [2021–2022 — Semantics separated from the wire format](2021-2022-semantics-vs-wire.md) — RFC 9110 semantics; HTTP/1.1, /2, /3 as separate carriers.
- [2023–June 2026 — Contracts expand to events and workflows](2023-2026-contracts-events-workflows.md) — AsyncAPI 3.x, OpenAPI 3.2, Arazzo 1.1, OAuth BCP, MCP, A2A.
