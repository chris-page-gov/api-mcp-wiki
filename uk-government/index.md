# UK government sources

UK policy, standards, research and registries referenced in the document.

- [Public Sector Equality Duty (Equality Act 2010, s.149)](equality-act-psed.md) — Due regard to eliminating discrimination and advancing equality.
- [legislation.gov.uk — API-first (2012)](legislation-gov-uk-api.md) — The UK's landmark API-first service; the API was the 'beating heart'.
- [GDS API technical and data standards (Feb 2018)](gds-api-standards.md) — Cross-government API guidance: REST default, OpenAPI specification-first.
- [Open Standards Board — OpenAPI 3 (2019)](open-standards-board-openapi3.md) — Formally recommended OpenAPI 3 for describing RESTful APIs.
- [Blueprint for modern digital government (Jan 2025)](blueprint-modern-digital-government.md) — Proposes a Digital Backbone, open APIs by default, and a 'once only' rule.
- [Algorithmic Transparency Recording Standard (ATRS)](atrs.md) — Mandatory across central government; tied to digital spend controls.
- [Cross-government API discovery (April 2025)](api-discovery-2025.md) — Found API readiness and maturity varied significantly and was generally low.
- [Data (Use and Access) Act 2025 — automated decisions (Arts 22A–22D)](duaa-2025-adm.md) — Replaces UK GDPR Art 22; safeguards for significant solely-automated decisions.
- [API Hub alpha (2025)](api-hub-alpha.md) — Identified demand for a refreshed catalogue, quality/security scores and developer tooling.
- [GOV.UK API Catalogue](api-catalogue.md) — Discovery registry; inclusion does not imply public accessibility.
- [DPIA & data protection by design (UK GDPR Arts 35, 25)](dpia-uk-gdpr.md) — High-risk processing needs a DPIA before go-live; privacy designed in.
